What is the .Net framework?

- The .NET framework supports an object-oriented approach that is used for building applications on windows. - It supports various languages like C#, VB.

What is an assembly?

An assembly is a file that is automatically generated by the compiler.

What is a garbage collector?

Garbage collector frees the unused code objects in the memory.

What is CLR?

CLR stands for common language run-time.
CLR manages the execution of programs written in any language (C#, VB.Net, F#) that uses the .NET Framework.

What is a Class?

A class is a blueprint of an object.
A class has fields, properties, methods.

What is an Object?

An object is an instance of a class through which we access the methods, fields, properties of that class. “New” keyword is used to create an object.

static

- static keyword can be used in properties/fields/methods to access them without creating object. (<>
- static keyword also can be used in static class for extension methods.

extension method?

- Extension methods enable you to add methods to existing types without creating a new derived type, recompiling, or otherwise modifying the original type.
- An extension method is a static method of a static class, where the "this" modifier is applied to the first parameter. The type of the first parameter will be the type that is extended.

Constructor?

A constructor is used to initialize the instance variables. Constructor name must be same as class name and it does not have a return type.

default constructor?

A default constructor is one without parameters

constant vs readonly

constant:
- It's mandatory to assign a value to constant variables
- Value is constant at compile time.
- We cannot change the value of a const variable throughout the entire program.
readonly:
- We can change during runtime or we can assign it at run time but only through the non-static constructor.

Polymorphism

- Polymorphism means many forms.
- Two Types
a) Compile time polymorphism also know as Overloading
b) Run time polymorphism also know as Overriding

late binding and early binding

Compile Time Polymorphism also known as Early Binding or Overloading
Run Time Polymorphism is also known as Late Binding or Overriding

overloading

same method but different no. of arguments or type of arguments

overriding

same method and same arguments. Methods in base class with virtual/abstract keywords can be overridden from derived class.
