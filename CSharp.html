extension method?

- Extension methods enable you to add methods to existing types without creating a new derived type, recompiling, or otherwise modifying the original type.
- An extension method is a static method of a static class, where the "this" modifier is applied to the first parameter. The type of the first parameter will be the type that is extended.

Constructor?

A constructor is used to initialize the instance variables. Constructor name must be same as class name and it does not have a return type.

default constructor?

A default constructor is one without parameters

constant vs readonly

constant:
- It's mandatory to assign a value to constant variables
- Value is constant at compile time.
- We cannot change the value of a const variable throughout the entire program.
readonly:
- We can change during runtime or we can assign it at run time but only through the non-static constructor.

What is the difference between Interface and Abstract Class in C#

- An abstract class can have non-abstract methods (concrete methods) while in case of interface, all the methods have to be abstract.
- An abstract class can declare or use any variables while an interface is not allowed to do so.
- In an abstract class, all data members or functions are private by default while in an interface all are public, we can’t change them manually.

What is enum in C#

An enum is a value type with a set of related named constants

What is the difference between a struct and a class in C#

Class and struct are both user-defined data types
Struct:
- The struct is a value type
- Struct can’t be inherited from other types
- Cannot create Constructor
Class:
- The class is a reference type
- Classes can be inherited from other classes

Value types(primitive) and Reference types?

- Basic data types int, char, bool, and long, which are value types.
- Classes and collections are reference types.

sealed classes

partial classes

boxing and unboxing

boxing:
Converting a value type data type to the object or to any interface.
int i = 10;
object obj = i;
unboxing:
Extract the value type from the object or interface
object obj = 10;
int i = (int)obj;

What is a Virtual Method in C#

- A virtual method is created in the base class that can be overridden in the derived class.
- It is used when a method's basic functionality is the same but sometimes more functionality is needed in the derived class.

What are the differences between IEnumerable and IQueryable

IEnumerable:
- It is the parent interface for all non-generic collections
- It is used for query the data from in-memory collections like List, Array etc.
IQueryable:
- It is used for query the data from database, suitable for linq to sql.
- Supports lazy loading so we can use in paging kind of scenarios

throw vs throw exception

Throw basically retains the stack trace.
Throw exception overwrites the stack trace.

String and StringBuilder

String:
- Immutable object
- Performance-wise, string is slow because it creates a new instance to override or change the previous value
StringBuilder:
- Mutable object
- Performance-wise StringBuilder is very fast because it will use the same instance of StringBuilder object to perform any operation like inserting a value in the existing string

ref vs out

ref:
- It is necessary the parameters should initialize before it pass to ref.
- bi-directional
out:
- It is necessary to initialize the value of a parameter before return.
- unidirectional

Pass by Reference

- The ref/out keyword indicates a value that is passed by reference
- When we pass parameters by reference the value of the parameter is changed, then the actual value stored at the memory location gets changed as no copy is getting created for the value passed as parameter.

Array vs ArrayList

Array:
- Array is strongly typed. This means that an array can have items of the same type only.
- The size of the array is fixed.
ArrayList:
- ArrayList can store any type of items.
- The size of the array is not fixed.

optional params?

Parameters are optional when a default value is specified to a particular argument.
If we do not pass any parameter to the optional arguments, then it takes its default value.
The default value of an optional parameter is a constant expression.
The optional parameters are always defined at the end of the parameter list.
